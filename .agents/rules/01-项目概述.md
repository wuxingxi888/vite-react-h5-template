# 项目概述

## 项目定位

一套面向 React 通用项目的 AI 落地模板，适用于 H5、管理后台、内容平台和中后台等单页应用场景。模板基于 TypeScript 严格模式开发，强调工程化规范、可执行的目录约束、统一的状态边界与可维护的组件化架构。

## 技术栈

| 领域 | 技术 | 版本 | 约束 |
|------|------|------|------|
| 构建工具 | Vite | ^7.1.2 | 强制使用 |
| UI 框架 | React | ^18.3.1 | 强制使用函数式组件 + Hooks |
| 语言 | TypeScript | ~5.8.3 | 强制使用严格模式，禁止 JavaScript 业务代码 |
| 路由 | React Router | 7.8.2 | 统一使用 Data Router 风格，路由按模块化配置 |
| 状态管理 | Zustand + Immer | 5.0.8 / 10.1.3 | 统一使用，禁止引入其他全局状态方案 |
| UI 组件库 | Ant Design Mobile | 5.39.0 | 交互组件优先基于该组件库构建 |
| 样式方案 | SCSS + UnoCSS | 1.92.1 / 66.5.1 | 原子样式与工程样式并用 |
| 包管理器 | pnpm | >=9 | 强制使用，禁止 npm / yarn |
| 图表库 | ECharts | 6.0.0 | 需要图表时统一使用 |
| 动画库 | Framer Motion | 12.23.13 | 复杂动效优先使用 |
| 工具函数库 | es-toolkit | 1.39.10 | 工具函数优先复用该库 |
| 事件通信 | mitt | 3.0.1 | 轻量事件总线场景可使用 |

## 模板基线

本模板统一采用以下目录与工程口径，后续规则与技能均以此为准：

- 页面目录：`src/views/`
- 路由目录：`src/routes/`
- 接口目录：`src/http/`
- 状态目录：`src/stores/`
- 共享类型目录：`src/types/`
- 通用组件目录：`src/components/`
- 页面局部样式与组件样式：优先使用 `index.module.scss`

## 适用边界

- 本模板面向 React 通用工程，不绑定 Next.js、React Server Components、Server Actions 或 React 19 专属能力
- 若项目后续采用 React 19、Next.js 或服务端组件架构，应在独立规则中显式扩展，不应直接覆盖本模板基线
