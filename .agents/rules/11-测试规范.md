# 测试规范

## 最小验证基线

- **基础门禁**：所有代码必须通过 TypeScript 类型检查和 ESLint 检查
- **关键逻辑验证**：新增功能必须包含必要的类型定义；关键业务逻辑应有可执行测试或明确可复现的验证步骤

## 推荐覆盖场景

- 路由守卫、默认重定向、404 兜底等路由行为
- store 的选择器、关键 action、持久化白名单等状态逻辑
- API 封装的错误分支、竞态处理或请求取消策略
- 组件关键交互状态，如 loading / empty / error / success 切换

## 验收要求

- 若暂时无法补自动化测试，至少应在任务说明中写出最小手工验证步骤
- 不得仅以“代码可运行”替代关键行为验证
