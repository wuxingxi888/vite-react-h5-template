# 编码规范

## TypeScript 规范

- 所有业务代码使用 TypeScript，禁止新增 JavaScript 业务文件
- 类型边界建议显式声明：组件 Props、Hooks 入参/返回值、公共函数、API 数据模型
- 局部变量与可推断场景优先使用类型推断，避免冗余类型注解
- 避免使用 `any`，确需使用时需说明原因并限制作用范围，优先 `unknown`
- 接口与类型命名统一使用 PascalCase

## React 与 Hooks 规范

- 使用函数式组件，避免类组件
- 遵守 Hooks 规则：Hook 仅在组件或自定义 Hook 顶层调用
- `useEffect` 依赖应完整；如有意省略依赖，需附注释说明原因
- 列表渲染使用稳定 `key`，避免随机值和临时时间戳
- 页面组件以编排为主，业务逻辑优先下沉到 hooks/子组件

## 性能与可维护性建议

- 避免在渲染阶段创建重量级对象/函数并层层透传
- 高频更新区域按需使用 `memo` / `useMemo` / `useCallback`
- 当 props 透传层级过深时，评估 context 或 store 方案
- 异步请求建议处理竞态（取消请求或忽略过期响应）
- 复杂函数建议拆分为更小的语义单元（建议单函数控制在可读范围内）

## 命名规范

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| 文件夹/路由 | kebab-case | `user-profile`、`login` |
| 变量/函数 | camelCase | `userList`、`fetchUserList` |
| 常量 | UPPER_SNAKE_CASE | `API_TIMEOUT` |
| 枚举类型 | PascalCase | `ApiMethod` |
| 枚举成员 | UPPER_SNAKE_CASE | `ApiMethod.GET` |
| 接口/类型/类 | PascalCase | `UserInfo`、`BaseResponse` |
| React 组件 | PascalCase | `UserProfile` |
| 自定义 Hooks | use 开头 | `useUserStore` |

## 函数命名规则

- **Props 回调**：`onXxx`（如 `onSubmit`、`onClose`）
- **组件内部处理函数**：`handleXxx`（如 `handleSubmit`、`handleDelete`）
- **布尔判断函数/变量**：`is/has/can/should` 开头
- **异步函数**：动词短语，必要时加 `Async` 后缀

## 导入与代码质量

- 使用 `@/` 别名导入 `src` 下模块
- 导入顺序：第三方依赖 -> 项目内部模块 -> 相对路径
- 避免循环依赖，公共能力按职责归档
- 遵循 ESLint / Prettier / Stylelint，避免无理由禁用规则

## 禁止项（精简）

- 禁止在 render 过程中直接发起请求或执行副作用
- 禁止直接修改 state（对象/数组原地改写）
- 禁止为“通过检查”而无说明地关闭 lint 规则
