## 浏览器工具打开页面指南

**核心原则**：验收必须以「实际页面效果 vs 设计稿」为准，必须使用浏览器工具查看目标页面。

### 优先使用当前环境中最稳定的浏览器工具

**操作步骤**：
1. 使用浏览器工具的导航能力前往目标 URL（例如 `http://localhost:5222/craft-sheet`）
2. 等待页面加载完成
3. 确认页面已加载完成、无报错，能够看到需要验收的区域

**工具说明**：
- 任意能打开页面、截取页面截图、获取页面快照（DOM/元素）的浏览器工具都可用。
- 若当前环境提供 Browser、Playwright 或等价工具，优先选择最稳定、最容易重复执行的一种。

### 浏览器工具不可用时的回退方式

**操作步骤**：
1. 选择当前环境可用的自动化浏览器、远程调试或截图方案
2. 前往目标 URL 并等待页面加载完成

**工具说明**：
- Playwright 等自动化浏览器工具通常可用于打开目标页面、截图（整页或元素）、获取页面快照、查找元素并执行点击/输入/滚动/表单提交等交互。
- 若需要访问受保护页面，可配合 cookies、localStorage 或模拟登录。

### 获取实际页面的可比对信息

**截图比对**：
- 优先用当前浏览器工具的截图能力对目标页面（或关键区域）截图
- 用途：与设计稿同区域截图并排对比

**元素/快照比对**：
- 优先用当前浏览器工具的快照能力获取页面结构、元素位置与尺寸
- 用途：与设计稿（.pen/Figma）中对应节点坐标、尺寸、样式逐项对比

**建议**：先做**整体或分区域截图**，与设计稿截图对比，发现差异后再用**元素快照**精确定位（如某块 padding、某字体大小）。

**验收时检查**：
- [ ] **按钮高度、宽度**：必须用浏览器工具实际查看渲染效果，不能仅凭代码推测
- [ ] **对齐效果**：必须用浏览器查看实际对齐效果
- [ ] **hover/active 状态**：必须用浏览器实际触发查看
- [ ] **响应式表现**：必须用浏览器在不同视口下查看
